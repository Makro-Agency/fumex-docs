Your Fumex Website — Owner's Guide

A plain-English guide to everything we built: every page, every feature, and exactly where to go in Shopify to change things yourself. No code required.

Theme: Makro CorePlatform: ShopifyBuilt by Makro AgencyUpdated June 2026

This guide walks through your site one topic at a time — pick anything from the menu on the left, or use Prev / Next (top right) to move through in order. For each page you'll see what it is, how it's built, and how to edit it.

Jump to the basics

💡 The golden rule

Editing content (text, images, products, prices, blog posts) is always safe — go ahead. Editing code (Online Store → Themes → Edit code) can break the design. When in doubt, change content in the visual editor and leave code to Makro.

Finding your way in Shopify

The three places you'll spend all your time. Everything lives in your admin at admin.shopify.com.

1. The Theme Editor — the look of your pages

The drag-and-drop visual editor: change banners, text, images, the order of things, colours and fonts.

📍 Where to find it

Online Store → Themes → Customize (on Makro Core). Use the page dropdown at the top to switch between Homepage, Product, Collection, etc.

2. Content — products, pages, blogs & menus

The actual "stuff": products & prices, written pages, blog articles, navigation menus, and the special Applications / Industries / OEM records (metaobjects).

📍 Where to find it

Left sidebar of your admin: Products, Content → Pages, Content → Blog posts, Content → Metaobjects, Content → Menus.

3. Theme Settings — site-wide design

Global choices applied everywhere: colours, fonts, logo, social links, cart style. See Colours & fonts.

✏️ A safe habit

The Theme Editor only goes live when you press Save. To experiment freely, ask Makro to set up a duplicate theme to play in.

Header & navigation

The bar at the top of every page.

Fixed to the top as visitors scroll. Left to right: the Fumex logo, the main menu (Products, Solutions, Help Center, Blog, About, Contact), a search icon, a cart icon, and the yellow Request Quote button. Products and Solutions open mega-menus — large dropdown panels with links and a promo block.

The links all come from Content → Menus. Here's how they map:

✏️ To change menu links

Content → Menus → open the relevant menu above → add/remove/reorder links. The mega-menu headings, promo images and the Request Quote button live in the header settings of the Theme Editor.

Footer

The dark-blue band at the bottom of every page.

A "Have Questions? Contact Us" prompt, a newsletter sign-up, columns of links (Products, Solutions, Help Center, About, Blog), and the copyright line.

✏️ How to edit it

Each column reuses a menu under Content → Menus — the same Products, Solutions, Help center, About and Blog menus the header uses (so editing one updates both places). The bottom strip (Search, "Your Privacy Choices") is the Footer menu. Headings, newsletter text and colours are in the footer settings of the Theme Editor.

The content engine

How your Applications, Industries and OEM pages build themselves — worth understanding.

Instead of hand-building dozens of near-identical pages, we use metaobjects. Think of a metaobject as a fill-in-the-blanks form: complete the fields, save, and a fully designed page appears at its own address. Edit the record and the page updates — every page stays consistent. You'll find them all under Content → Metaobjects.

✏️ Add a new application page — step by step

1. Content → Metaobjects → Individual Application Pages → Add entry.
2. Fill in the fields:
  • Instance Name (internal label, e.g. "Laser Welding")
  • Title, Hero Image, Short Description, Main Description
  • Rich text / Image with text Reference (the body blocks)
  • Product Reference heading + Product Reference (recommended products)
  • SEO Title & SEO description
3. Save — the page is created automatically.
4. To list it on the hub, open the Applications Landing Page metaobject and add your new entry to its Applications list.

💡 Industries work identically

Same steps under Individual Industry Pages (it adds a Sub title and an Other Industries list), then add it to the Industries Landing Page record.

⚠️ Keep the fields filled

The design relies on these fields — leaving an image or heading blank can leave a gap. Fill every field the existing entries use, and don't change the Instance Name of records the menus already point to.

Colours & fonts

The site-wide design settings.

Brand styling is controlled centrally so the whole site stays consistent:

  • Fonts — headings & body in DM Sans; labels, buttons & menus in Roboto Mono (the uppercase technical text).
  • Colours — yellow primary buttons (#F7D400), blue accents (#2e69fd), dark backgrounds, blue badges.
  • Shape — rounded corners (10px) and pill-shaped buttons.
  • Logo, favicon, social links, dark-mode — all set here too.
📍 Where to find it

Theme Editor → Theme settings (gear icon, bottom of the left panel) → Colors, Typography, Logo.

⚠️ Go gently

These affect every page at once. Change one thing at a time and preview before saving.

How we built it

A short, non-technical summary of the approach.

We built Fumex a custom Shopify theme ("Makro Core") from the ground up rather than an off-the-shelf template. Pages are assembled from reusable sections so they can be rearranged in the visual editor without touching code.

The clever part is the content architecture. Fumex has many applications, industries and product lines. Instead of building each page by hand, we created metaobjects — structured records that automatically generate consistent, on-brand pages. Add a record, get a page. The site scales as Fumex grows, with no developer needed for new applications or industries.

We also connected the tools your team already uses — HubSpot for leads, Gorgias for support, ReCharge for subscriptions — and built conversion features like the Product Selector Quiz to guide visitors to the right system.
